Insertion Sort

Efficient for small amount of elements

Element compare with all previous elements and inserted at the proper position. In iteration i, swap a[i] with each larger entry to its left. Scans from left to right. To maintain algorithm invariants: Move the pointer to the right, Moving from right to left, exchange a[i] with each larger entry to its left, To sort a randomly-ordered array with distinct keys, insertion sort uses ~ ¼ N 2 compares and ~ ¼ N 2 exchanges on average. Expect each entry to move halfway back. Best case. If the array is in ascending order, insertion sort makes N- 1 compares and 0 exchanges. Worst case. If the array is in descending order (and no duplicates), insertion sort makes ~ ½ N 2 compares and ~ ½ N 2 exchanges. An inversion is a pair of keys that are out of order. An array is partially sorted if the number of inversions is ≤ c N. For partially-sorted arrays, insertion sort runs in linear time. Number of exchanges equals the number of inversions

O(n²) - for every element (started from the second) is performed compare operation for elements with smaller index

Type : Algorithm of stable sorting

Merge Sort

Divide and Conquer.

Divede into two parts and sort. Merge to parts in one. Take element from one sequence and from another and compare = take the smallest and put it to the array. Divide array recursively till it will be only one element. Perform merge till one sequence

O(n log n)

Algorithm of stable sorting

Bubble Sort

Not efficient but very popular

It works by repeatedly swapping adjacent element that are out of order

O(n²) - begins from the end of a list, Compares two adjacent elements

Algorithm of stable sorting

Selection Sort

Looks for the smallest element and swap it. In iteration i, find index min of smallest remaining entry. Swap a[i] and a[min]. Scans from left to right. To maintain algorithm invariants: Move the pointer to the right, Identify index of minimum entry on right, Exchange into position. Selection sort uses (N– 1) + (N– 2) + ... + 1 + 0 ~ N 2 / 2 compares and N exchanges. Running time insensitive to input. Quadratic time, even if input is sorted. Data movement is minimal. Linear number of exchanges.

O(n²) - begins from the end of a list, Compares two adjacent elements

Algorithm of unstable sorting

Shell Sort

Move entries more than one position at a time by h-sorting the array. A g-sorted array remains g-sorted after h-sorting it. The worst-case number of compares used by shellsort with the 3x+1 increments is O(N 3/2). Fast unless array size is huge (used for small subarrays). Tiny, fixed footprint for code (used in some embedded systems). Hardware sort prototype.
